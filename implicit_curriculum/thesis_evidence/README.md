# Thesis Evidence Archive

This folder is a durable, incrementally extended evidence layer for the thesis. It is intentionally parallel to `src/`, `tests/`, and `configs/` so it survives code-version updates.

The rule is:

- `results/` can be overwritten during active experimentation.
- `thesis_evidence/` should be append-only except for clearly documented corrections.
- Every result that may support a thesis claim should be summarized here with claim scope, source files, and whether it is final, pilot, or diagnostic.

## Folder contents

- `EXPERIMENT_LOG.md` — chronological summary of what has been run and what was learned.
- `CLAIMS_AND_EVIDENCE.md` — claim-level table linking possible thesis claims to supporting runs and caveats.
- `RESULTS_REGISTRY.csv` — machine-readable registry of thesis-relevant runs.
- `FIGURES_TABLES_TODO.md` — planned thesis figures/tables and required source files.
- `NEXT_HYPOTHESES.md` — hypotheses generated by current results for follow-up experiments.
- `results_summaries/` — compact copied source artifacts, not large checkpoints.
- `tables/` — curated summary CSVs for plotting/table generation.
- `figures_plans/` — figure specification notes.

## Evidence status labels

- `green`: result supports a scoped claim or next-step decision.
- `yellow`: result is informative but not sufficient for a strong claim.
- `red`: result blocks the intended claim or indicates redesign is needed.
- `diagnostic`: engineering/instrumentation result, not thesis evidence by itself.

## Claim tiers

- `Tier 1`: causal claim in controlled settings only.
- `Tier 2`: phenomenon-level/predictive claim in controlled settings.
- `Tier 3`: observational LLM corroboration only.

No result currently supports a real-LLM causal claim.
